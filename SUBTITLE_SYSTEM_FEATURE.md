## 📋 功能概述

本次实现完成了完整的字幕系统增强,包括 YouTube 字幕获取、多格式字幕解析、字幕编辑器等核心功能,大幅提升了应用的实用性和灵活性。

## ✨ 核心功能

### 1. YouTube 字幕获取增强

#### 已有功能 (通过 youtube-transcript 库)
- 自动获取 YouTube 视频字幕
- 支持多语言字幕
- 通过 `/api/transcript` API 路由提供服务

#### 新增功能 (lib/youtube-api.ts)

**获取可用字幕列表**
```typescript
getAvailableCaptions(videoId: string): Promise<Array<{
  language: string;
  languageCode: string;
  name: string;
  isAutoGenerated: boolean;
}>>
```

**直接获取 YouTube 字幕**
```typescript
getYouTubeCaptions(
  videoId: string,
  languageCode: string = 'en'
): Promise<YouTubeCaption[]>
```
- 使用 YouTube timedtext API (公开接口)
- 支持自动生成字幕 (ASR)
- 自动解析 JSON3 格式

**检测字幕可用性**
```typescript
hasAvailableCaptions(
  videoId: string,
  languageCode?: string
): Promise<boolean>
```
- 快速检测视频是否有字幕
- 支持多语言检测

**字幕格式转换**
```typescript
convertToAppSubtitles(
  youtubeCaptions: YouTubeCaption[]
): Array<{index, start, end, text}>
```

### 2. 字幕解析器 (lib/subtitle-parser.ts)

#### 支持的格式
- **SRT** (SubRip Text) - 最常用的字幕格式
- **VTT** (WebVTT) - Web 标准字幕格式
- **JSON** - 自定义 JSON 格式

#### 核心功能

**解析字幕文件**
```typescript
parseSubtitleFile(content: string, filename: string): ParsedSubtitle[]
```
- 自动检测文件格式
- 支持三种格式解析
- 错误处理和验证

**导出字幕**
```typescript
exportToSRT(subtitles: ParsedSubtitle[]): string
exportToVTT(subtitles: ParsedSubtitle[]): string
exportToJSON(subtitles: ParsedSubtitle[]): string
```

**字幕验证**
```typescript
validateSubtitles(subtitles: ParsedSubtitle[]): {
  valid: boolean;
  errors: string[];
}
```
验证规则:
- 开始时间不能为负数
- 结束时间必须大于开始时间
- 文本不能为空
- 检测时间重叠

**字幕优化**
```typescript
// 合并相邻的短字幕
mergeShortSubtitles(
  subtitles: ParsedSubtitle[],
  minDuration: number = 1.5
): ParsedSubtitle[]

// 调整时间偏移
adjustSubtitleTiming(
  subtitles: ParsedSubtitle[],
  offsetSeconds: number
): ParsedSubtitle[]
```

### 3. 字幕编辑器 (components/subtitle-editor.tsx)

#### 用户界面

**工具栏功能**
- 📤 上传字幕文件 (SRT, VTT, JSON)

**验证和反馈**
- 实时验证字幕数据
- 成功/错误消息提示
- 字幕数量统计
- 时间格式化显示

#### 技术特性

### 文件结构

```
lib/
├── youtube-api.ts              # YouTube API 增强

components/
└── subtitle-editor.tsx         # 字幕编辑器组件

app/api/
└── transcript/route.ts         # 字幕 API 路由
```

### 数据流

```
YouTube 视频
    ↓
1. 获取字幕列表 (getAvailableCaptions)
    ↓
2. 下载字幕 (getYouTubeCaptions 或 fetchYouTubeSubtitles)
    ↓
3. 解析字幕 (parseSubtitleFile)
    ↓
4. 验证字幕 (validateSubtitles)
    ↓
5. 编辑字幕 (SubtitleEditor)
    ↓
6. 导出字幕 (exportToSRT/VTT/JSON)
```

### 字幕格式示例

**SRT 格式**
```
1
00:00:00,000 --> 00:00:03,000
Hello, welcome to the video.

2
00:00:03,500 --> 00:00:06,000
Today we'll learn about subtitles.
```

**VTT 格式**
```
WEBVTT

00:00:00.000 --> 00:00:03.000
Hello, welcome to the video.

00:00:03.500 --> 00:00:06.000
Today we'll learn about subtitles.
```

**JSON 格式**
```json
[
  {
    "index": 1,
    "start": 0,
    "end": 3,
    "text": "Hello, welcome to the video."
  },
  {
    "index": 2,
    "start": 3.5,
    "end": 6,
    "text": "Today we'll learn about subtitles."
  }
]
```

## 📝 使用示例

### 场景 1: 上传自定义字幕

1. 用户准备了一个 SRT 字幕文件
2. 打开字幕编辑器
3. 点击"上传字幕"按钮
4. 选择 SRT 文件
5. 系统自动解析并显示字幕
6. 用户可以编辑、调整时间
7. 导出为 VTT 格式用于 Web 播放

### 场景 2: 调整字幕时间

1. 用户发现字幕延迟 2 秒
2. 打开字幕编辑器
3. 在"时间偏移"输入 `-2`
4. 点击"应用"
5. 所有字幕时间自动调整
6. 保存或导出调整后的字幕

### 场景 3: 合并短字幕

1. 用户上传的字幕有很多短句
2. 点击"合并短字幕"按钮
3. 系统自动合并时长 < 1.5 秒的相邻字幕
4. 字幕数量减少,阅读体验提升
5. 导出优化后的字幕

### 场景 4: 从 YouTube 获取字幕

1. 用户想要某个 YouTube 视频的字幕
2. 系统调用 `getYouTubeCaptions(videoId, 'en')`
3. 自动下载并解析字幕
4. 转换为应用内格式
5. 用户可以在编辑器中查看和编辑

## 🔄 集成方式

### 在视频学习界面中使用

```typescript
import { SubtitleEditor } from '@/components/subtitle-editor';
import { getYouTubeCaptions, convertToAppSubtitles } from '@/lib/youtube-api';

// 获取 YouTube 字幕
const captions = await getYouTubeCaptions(videoId, 'en');
const subtitles = convertToAppSubtitles(captions);

// 打开编辑器
<SubtitleEditor
  videoId={videoId}
  initialSubtitles={subtitles}
  onSave={(edited) => {
    // 保存编辑后的字幕
    saveSubtitles(videoId, edited);
  }}
  onClose={() => setShowEditor(false)}
/>
```

### 在管理后台中使用

```typescript
// 管理员上传字幕
<SubtitleEditor
  videoId={videoId}
  onSave={(subtitles) => {
    // 保存到数据库或 localStorage
    saveToDatabase(videoId, subtitles);
  }}
/>
```
