**修复文件**:
- `components/video-learning-interface-sync.tsx` (第 84-90 行)

### 2. 保存单词报错"缺少视频信息" ✅

**问题描述**:
- 点击单词后，在弹窗中点击保存按钮
- 显示错误: "无法保存：缺少视频信息"

**根本原因**:
- `VideoLearningInterface.tsx` 中调用 `WordPopup` 组件时
- 缺少必需的参数: `context`, `timestamp`, `videoId`
- `WordPopup` 的 `handleSave` 函数需要这些参数才能保存

**修复方案**:
- 在 `VideoLearningInterface.tsx` 第 241-247 行
- 添加缺失的三个参数传递

**修复代码**:
```typescript
<WordPopup
  word={selectedWord.word}
  position={{ x: selectedWord.x, y: selectedWord.y }}
  onClose={() => setSelectedWord(null)}
  context={selectedWord.context}      // ✅ 新增
  timestamp={selectedWord.timestamp}  // ✅ 新增
  videoId={videoId}                   // ✅ 新增
/>
```

### 4. 字幕编辑器关闭不了 ✅

**问题描述**:
- 打开字幕编辑器后，点击 X 按钮无法关闭
- 点击背景遮罩也无法关闭

**根本原因**:
- 模态框背景遮罩没有 `onClick` 事件
- 内容区域没有阻止事件冒泡

**修复方案**:
- 在背景遮罩 `div` 上添加 `onClick={() => setShowSubtitleEditor(false)}`
- 在内容区域 `div` 上添加 `onClick={(e) => e.stopPropagation()}`
- 现在支持两种关闭方式：点击 X 按钮或点击背景

**修复文件**:
- `components/video-player/VideoLearningInterface.tsx` (第 253-260 行)

### 5. 缺失的 PWA 图标文件 ✅

**问题描述**:
- 开发服务器日志显示多个 404 错误
- `icon-192x192.png`, `icon-512x512.png`, `favicon.ico` 不存在

**根本原因**:
- PWA manifest.json 引用了这些图标
- 但 `public/` 目录中没有这些文件

**修复方案**:
- 使用 Node.js 创建最小有效的 PNG 和 ICO 文件
- 作为临时占位符，避免 404 错误
- 后续可以替换为正式的品牌图标

**创建的文件**:
- `public/icon-192x192.png` (67 bytes)
- `public/icon-512x512.png` (67 bytes)
- `public/favicon.ico` (1.1 KB)
