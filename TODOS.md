  - **依赖**：词汇保存功能
  - **验收标准**：

### 用户系统（后端）
- [ ] **技术选型决策**
  - **描述**：确定后端技术栈
  - **工作量**：1 小时（调研 + 讨论）
  - **选项**：
    - **Option A**: Supabase（推荐）
      - 优点：开箱即用的认证、数据库、实时订阅

## 📝 笔记

### 开发原则
1. **用户至上**：每个功能都要问"这对用户有什么价值？"
2. **快速迭代**：先做 MVP，再优化
3. **数据驱动**：用统计数据指导功能优先级
4. **开源精神**：代码质量要经得起审查

### 技术债务追踪
- ⚠️ **无测试覆盖**：需要添加单元测试和 E2E 测试
- ⚠️ **无错误监控**：需要集成 Sentry
- ⚠️ **性能未优化**：需要代码分割、懒加载
- ⚠️ **无 CI/CD**：需要设置 GitHub Actions

### 决策记录
- **2026-04-09**：决定使用 localStorage 作为 Phase 1 的数据存储方案，Phase 2 再迁移到云端
- **2026-04-09**：决定使用 MyMemory 作为翻译 API（免费，无需 API key）
