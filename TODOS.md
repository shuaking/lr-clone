## 📝 笔记

### 开发原则
1. **用户至上**: 每个功能都要问"这对用户有什么价值？"
2. **快速迭代**: 先做 MVP,再优化
3. **数据驱动**: 用统计数据指导功能优先级
4. **代码质量**: 保持测试覆盖率,遵循最佳实践

### 技术债务追踪
- ⚠️ **测试覆盖率不足**: 当前有测试框架,需要增加测试用例（目标 >80%）
- ⚠️ **无错误监控**: 需要集成 Sentry
- ⚠️ **无 CI/CD**: 需要设置 GitHub Actions
- ⚠️ **无 E2E 测试**: 需要 Playwright 或 Cypress

- **2026-04-10**: Phase 1 核心功能达到 100% 完成（所有遗留功能已实现）

- **2026-04-10**: 虚拟滚动实现完成，使用自定义 VirtualScroll 组件
- **2026-04-10**: 主题系统完善，支持 light/dark/system 三种模式
- **2026-04-10**: 下一步优先级: 性能优化（PWA、字幕预加载）→ 多语言支持
